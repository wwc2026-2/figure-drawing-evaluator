# 参考照片与人物线稿骨架对比评分系统

本工程是一个网页端原型，用于“参考照片”和“自己画的人物线稿”之间的人体结构对比评分。

## 核心功能

- 上传参考照片
- 上传自己的线稿
- 自动识别参考照片人体骨架
- 尝试自动识别线稿骨架
- 在线稿图上标注关键骨骼点
- 支持拖动关键点进行手动校准
- 根据参考照片与线稿关键点进行对比评分
- 输出六边形雷达图
- 输出教学诊断建议
- 支持导出文本评分报告

## 评分维度

- 比例
- 结构
- 动态
- 线条
- 造型
- 完整度

## 关键点

当前标注 13 个主要人体结构点：

- 头部
- 左肩、右肩
- 左肘、右肘
- 左腕、右腕
- 左胯、右胯
- 左膝、右膝
- 左踝、右踝

cd figure-drawing-compare-pose

```bash

```

浏览器打开终端显示的地址，例如：

```text
http://localhost:5173/
```

## 使用流程

1. 上传参考照片。
2. 上传自己的线稿。
3. 点击“开始对比评分”。
4. 系统会识别参考照片骨架，并尝试识别线稿骨架。
5. 如果线稿识别不准确，可直接拖动线稿上的关键点校准。
6. 校准后系统会自动重新计算评分。
7. 查看雷达图、分项得分和诊断建议。
8. 可点击“导出报告”保存文本报告。

## 重要说明

MediaPipe Pose 对真实人体照片识别较稳定，但对线稿、动漫人物、夸张角色、铅笔稿识别可能不稳定。

因此本工程将 MediaPipe Pose 作为“自动辅助标点”，并增加了“手动拖动关键点修正”功能。即使线稿自动识别失败，也可以继续完成骨架标注与评分。

如果模型加载失败，请检查网络是否可以访问：

```text
https://cdn.jsdelivr.net
https://storage.googleapis.com
```

## 修正版说明

如果点击“开始对比评分”出现“分析失败”，常见原因不是图片格式本身，而是 MediaPipe 模型或 wasm 文件无法加载。

本修正版做了容错处理：

- 参考照片骨架识别失败时，自动生成默认骨架；
- 线稿骨架识别失败时，自动套用参考骨架作为初始标点；
- 用户可以拖动线稿关键点完成手动校准；
- MediaPipe 不可用时，仍可继续进行线稿标注与评分；
- 报错提示会显示更具体的排查方向。

建议优先使用 Chrome 或 Edge 浏览器运行。


## 中国大陆可访问 MediaPipe 模型库说明

本版本新增“模型源数据库”，不再强依赖 `storage.googleapis.com`。

### 推荐方案：项目本地模型库

将 MediaPipe Pose 模型文件放到：

```text
public/mediapipe/models/pose_landmarker_full.task
```

运行：

```bash
npm install
npm run setup:mediapipe
npm run dev
```

其中 `npm run setup:mediapipe` 会把 `@mediapipe/tasks-vision` 包里的 wasm 文件复制到：

```text
public/mediapipe/wasm
```

浏览器加载时会优先使用：

```text
/mediapipe/wasm
/mediapipe/models/pose_landmarker_full.task
```

这样部署到学校服务器、学院服务器或本地电脑后，中国大陆访问时不需要再访问 Google 模型源。

### 学校服务器 / 阿里云 OSS / 腾讯云 COS

也可以把以下目录部署到国内可访问服务器：

```text
mediapipe/wasm
mediapipe/models/pose_landmarker_full.task
```

然后在 `.env` 中配置：

```bash
VITE_MEDIAPIPE_WASM_PATH=https://your-domain.com/mediapipe/wasm
VITE_MEDIAPIPE_POSE_MODEL_PATH=https://your-domain.com/mediapipe/models/pose_landmarker_full.task
VITE_ALLOW_GOOGLE_MEDIAPIPE_FALLBACK=false
```

### 模型源配置文件

模型源数据库文件位于：

```text
public/mediapipe/model-database.json
src/lib/modelSources.js
```

默认加载顺序：

```text
1. .env 自定义模型库
2. 项目本地模型库 /mediapipe
3. Google 官方兜底源
```

如果所有模型源都失败，系统不会中断，而是自动切换为“手动骨架标注模式”。


## 手动校准评分修正版说明

本版本修复了“线稿未自动识别时分数不对”的问题。

旧版本逻辑：

```text
线稿识别失败
↓
自动套用参考骨架作为线稿骨架
↓
直接评分
↓
两套骨架过于接近，分数容易虚高
```

新版逻辑：

```text
线稿识别失败
↓
套用参考骨架只作为初始标点
↓
不直接生成分数
↓
提示用户拖动线稿关键点完成校准
↓
校准后重新计算有效分数
```

同时，参考照片骨架也支持拖动校准。即使参考照片无法自动识别，也可以通过手动拖动关键点完成对比评分。

推荐使用流程：

1. 上传参考照片。
2. 上传线稿。
3. 点击“开始识别 / 评分”。
4. 如果参考照片或线稿未自动识别，拖动对应图片上的关键点。
5. 确保头、肩、肘、腕、胯、膝、踝都贴合图像。
6. 系统自动重新计算分数。
7. 导出评分报告。


## 线稿不显示骨架线版本说明

本版本根据教学使用需求做了调整：

1. 线稿区不再显示骨架连线，避免遮挡学生线稿；
2. 线稿区只显示可拖动的关键点，包括头、肩、肘、腕、胯、膝、踝；
3. 参考照片区仍保留骨架线，便于观察标准姿态；
4. 评分算法增强了问题定位，不只给出分数，还会指出比例、结构、动态和关键点偏差；
5. 线稿识别失败后不会直接评分，必须完成关键点校准后再计算有效分数。

推荐流程：

```text
上传参考照片
上传线稿
开始识别 / 评分
校准参考照片骨架
校准线稿关键点
查看分数与主要问题定位
根据问题修改线稿
```


## 线稿纯净显示版说明

本版本根据最新要求做了以下修改：

1. 线稿区不套用参考骨架；
2. 线稿区不显示骨架连线；
3. 线稿区不显示可拖动关键点；
4. 线稿只显示原图；
5. 参考照片区仍可识别或校准骨架；
6. 评分采用“线稿图像评分模式”，依据线条、外轮廓、主体占比、完整度和参考姿态风险进行诊断；
7. 诊断建议会明确说明问题来源，不再把线稿评分伪装成精确骨架点对比。

注意：由于线稿区不再提供关键点，系统无法进行“参考骨架点 vs 线稿骨架点”的精确几何对比。比例、结构、动态分属于线稿图像特征诊断分，适合教学辅助，不等同于专业人体姿态关键点误差评分。


## 参考图基准评分版说明

本版本进一步调整为“线稿评分要根据参考图来”。

核心变化：

1. 线稿区仍然只显示原图；
2. 线稿区仍然不显示骨架、不显示可拖动关键点、不套用参考骨架；
3. 评分逻辑改为“参考图基准评分模式”；
4. 系统先从参考图提取姿态与构图特征（如人体占画面比例、外轮廓宽高比、动作动态强度等）；
5. 再将这些特征作为目标，用来评价线稿的比例、结构、动态、造型等分项；
6. 因此分数不再是通用固定阈值，而是围绕参考图生成。

需要注意：

- 因为线稿不显示关键点，所以当前版本仍不是“参考骨架点 vs 线稿骨架点”的精确几何误差评分；
- 但它已经实现“线稿评分根据参考图来”的目标，更适合你现在的界面要求。


## 问题精确指出版说明

本版本在“参考图基准评分”的基础上，进一步增强了问题诊断。

新增能力：

1. 明确指出问题名称，例如：
   - 人物画得偏小
   - 人物外轮廓偏宽 / 偏窄
   - 动作张力不足
   - 线条碎且反复描线较多
   - 结构线不足
   - 人物主体不够完整

2. 每个问题都包含：
   - 问题类别
   - 问题说明
   - 修改建议

3. 结果页新增“问题精确指出”模块，更适合教学反馈与学生修改。
